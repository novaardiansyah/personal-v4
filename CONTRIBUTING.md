# Contributing to Personal Filament v4 Admin Panel

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Check if the bug has already been reported.** Search the [issues](https://github.com/novaardiansyah/personal-v4/issues) to see if someone else has already reported it.
- **Open a new issue.** Clearly describe the issue, including steps to reproduce it if possible. Include information about your environment (PHP version, OS, etc.).

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

- **Check the existing issues.** See if someone else has already suggested the enhancement.
- **Open a new issue.** Provide a clear title and description of the suggested enhancement. Explain why it would be useful.

### Pull Requests

- **Fork the repository.** Create your own fork of the project.
- **Create a new branch.** Use a descriptive name for your branch (e.g., `fix/typo-in-readme` or `feat/add-new-dashboard-widget`).
- **Follow the coding standards.** Ensure your code follows the existing style and standards of the project (Laravel and Filament best practices).
- **Commit your changes.** Use clear and descriptive commit messages.
- **Submit a pull request.** Provide a clear description of your changes and why they are necessary.

## Coding Standards

- **Laravel Best Practices**: Follow the official [Laravel documentation](https://laravel.com/docs) and community standards.
- **Filament PHP**: Adhere to [Filament documentation](https://filamentphp.com/docs) for all admin panel components and logic.
- **PHP**: Use modern PHP features where applicable, targeting PHP 8.2+.
- **Clean Code**: Ensure your code is readable and maintainable.

## Getting Setup

1. Fork the repo and clone it locally.
2. Install dependencies: `composer install` and `npm install`.
3. Copy `.env.example` to `.env` and configure your environment.
4. Run migrations: `php artisan migrate`.
5. Start development!
